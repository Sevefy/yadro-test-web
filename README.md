# Тестовое задание от команды разработки внутренних продуктов на стажировку YADRO Импульс

Веб-приложение для работы с API randomdatatools.ru. Позволяет загружать, хранить и отображать информацию о пользователях с поддержкой пагинации.

## Cтек технологий
- FastAPI - современный веб-фреймворк
- SQLAlchemy + aiosqlite - асинхронная ORM и легковесная БД, которая не требует отдельного сервера
- Jinja2 - шаблонизатор для SSR
- Pydantic - валидация данных и схемы API
- pytest + pytest-asyncio + httpx - тестирование асинхронных приложений
- Docker + docker-compose - контейнеризация для упрощения развертывания приложения

## Архитектурные подходы
- Repository pattern - изоляция логики БД
- Dependency injection - инъекция зависимостей (пагинация и сессия БД)
- Асинхронная архитектура

## Требования
- Python 3.11+ (тестирование и разработка проводилась на 3.11)
- Docker, docker-compose

## Конфигурация

Создайте файлы `.env` и `.env.prod` в корне проекта:
```env
SERVER_HOST=0.0.0.0
SERVER_DEBUG=True/False
```
При запуске с SERVER_DEBUG=True запускается uvicorn с reload=True.

При запуске с SERVER_DEBUG=False запускается uvicorn с workers=4.

## Запуск
Для развертывания необходимо склонировать репозиторий и создать конфигурационные файлы описанные в "Конфигурация"
### Запуск через docker
1. Запустить контейнер (настройки будут подхвачены из .env.prod)

`docker compose up -d --build`

2. Приложение будет доступно по адресу: `http://localhost:8000`

### Локальный запуск

При локальном запуске будет использоваться файл `.env`

1. Создайте виртуальное окружение
```fish
python -m venv .venv
``` 
2. Активируйте виртуальное окружение
```fish
# для оболочки fish
source ./.venv/bin/activate.fish
```
```bash
#Для windows
./.venv/Scripts/activate
```
3. Запустить приложение
```
python -m app.main
```
4. Приложение будет доступно по адресу: `http://localhost:8000`

## API документация

После запуска приложения доступна автоматическая документация:
- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

## Основные "Ручки" (endpoints)

### SSR
- `GET /` - главная страница с таблицей пользователей (с пагинацией)
- `GET /{user_id}` - страница конкретного пользователя
- `GET /random` - страница со случайным пользователем из БД

### REST API

- `GET /api/v1/users` - список пользователей (с пагинацией)
- `GET /api/v1/users/{user_id}` - информация о конкретном пользователе 
- `GET /api/v1/users/random` - информация о случайном пользователе из БД
- `POST /api/v1/users/load_from_api` - загрузить новых пользователей с randomdatatools (в body нужно передать count)

## Тестирование

Проект покрыт тестами с использованием pytest и pytest-asyncio

### Запуск тестов
```bash
pytest -v
```

## Ограничения

- База данных сбрасывается при каждом запуске - по требованию задания, при старте сервера таблицы пересоздаются и после загружаются 1000 случайных пользователей из randomdatatools
- При запросах к randomdatatools могут приходить дубликаты данных, обработано через повторный запрос
- При пересоздании docker контейнера данные теряются (нет volume)
